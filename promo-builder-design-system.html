5. Template Layout Spec

실제 페이지 조립 기준
Template 4 Normal

운영 안정성이 가장 중요한 기본 랜딩입니다. 디자인 자유도는 낮지만, 리전 담당자가 콘텐츠를 교체해도 깨지지 않는 구조가 핵심입니다.

  • Hero는 문구와 CTA를 우선하고 이미지는 보조로 사용
  • Step Bar는 3단계 고정 또는 2-4단계 확장 가능
  • Image/Text Row는 반복 섹션으로 사용 가능
  • Terms와 Footer는 hide-on-empty가 아니라 정책 필수로 유지
promo_001 Bonus

보너스, 가입, 코드 사용 여부를 명확히 보여주는 전환형 랜딩입니다. Bonus Code 정책과 Affiliate Q-TAG 규칙의 영향을 직접 받습니다.

  • Hero에 Offer value, CTA, Bonus Code/No Code 메시지 노출
  • 가입 단계와 보너스 수령 조건을 가까운 위치에 배치
  • Bonus Code 사용 시 Tracker mapping과 eligibility 고지 필요
  • Affiliate 유입 시 brand_name, site_url sanitization 확인
promo_002 Casino

시각 임팩트와 동적 모듈이 필요한 카지노/슬롯 캠페인형입니다. 관리 통제가 어려운 요소는 옵션화하고 Rule Base로 제한합니다.

  • Dark Hero와 게임 비주얼을 첫 화면 핵심으로 사용
  • Community, Winners, Rewards는 옵션 모듈로 분리
  • Winners는 API 또는 mock 데이터 상태를 명확히 표시
  • Footer는 Widget과 license_section 정책을 유지

Terms / Footer Policy

모든 템플릿에서 Footer와 정책 영역은 디자인 장식이 아니라 운영 안전장치입니다. 빌더는 이 영역을 삭제 옵션으로 제공하지 않고, 문구와 표시 조건만 조절해야 합니다.

  • Responsible Gaming 문구와 연령 제한 표시
  • License section 또는 Widget footer 연결
  • Bonus Code 사용 시 bonus/tracker/eligibility 조건
  • Affiliate 사용 시 Q-TAG, brand name, sanitized site_url
  • 빈 데이터가 있어도 footer는 유지, 동적 모듈만 fallback 처리

8. Rule Base Design Constraints

디자인 통제 규칙
공통 고정 규칙
  • Hero, CTA, Terms, Footer는 모든 템플릿에서 필수
  • CTA는 Hero 첫 화면에 1회 이상 노출
  • 모바일에서 Hero는 단일 컬럼으로 전환
  • 텍스트는 컨테이너를 넘지 않도록 줄바꿈 허용
조건부 정책 규칙
  • Bonus Code 활성화 시 code, tracker, eligibility 검증
  • Affiliate 활성화 시 Q-TAG와 brand_name 확인
  • site_url은 직접 노출 전 sanitize 상태 필요
  • Winners API 의존 구간은 empty state 필수
디자인 자유도 제한
  • Template 4는 레이아웃 변형보다 콘텐츠 안정성 우선
  • promo_001은 보너스 메시지와 약관 근접 배치
  • promo_002의 motion/custom CSS는 옵션으로 격리
  • 푸터/라이선스/Responsible Gaming은 숨김 금지
