## 2. 문제 정의

현재 고객은 앱에서 가까운 매장, 현재 구매 가능한 상품, 자신에게 맞는 할인 혜택을 한 번에 찾기 어렵다. 매장별 재고와 쿠폰 정보가 분산되어 있어 탐색 시간이 길고, 매장 방문 전환과 쿠폰 사용률이 낮다. 운영팀은 캠페인 성과를 국가/매장/상품별로 비교하기 어렵고, 고객 세그먼트별 맞춤 캠페인을 빠르게 실행하기 어렵다.

## 3. 해결 방향

고객이 앱 또는 모바일 웹에 접속하면 AI 쇼핑 어시스턴트가 위치, 언어, 관심 카테고리, 구매 이력, 최근 본 상품, 매장 재고, 진행 중인 쿠폰 캠페인을 종합해 추천을 제공한다. 추천 결과에는 가까운 매장, 추천 상품, 혜택 쿠폰, 예상 이동 시간, 매장 픽업 가능 여부를 함께 표시한다. 고객은 추천 쿠폰을 저장하고 매장 또는 온라인 결제에서 사용할 수 있어야 한다.

## 4. 핵심 사용자

- 신규 고객: 첫 구매 혜택과 가까운 매장을 빠르게 찾고 싶다.
- 재방문 고객: 자신의 취향에 맞는 상품과 쿠폰을 추천받고 싶다.
- 오프라인 매장 방문 고객: 현재 위치 주변에서 재고가 있는 매장을 찾고 싶다.
- 캠페인 운영자: 국가/매장/상품별 쿠폰 성과를 분석하고 싶다.
- 시스템: 추천 점수, 재고, 쿠폰 발급 가능 여부를 실시간으로 계산해야 한다.

## 8. 현지화 및 법무 요구사항

- 언어 우선순위: 한국어 P0, 영어 P0, 일본어 P1
- 번역 방식: 한국어/영어는 전문 번역사 검수, 일본어는 기계번역 후 현지 담당자 검수
- 날짜 형식: KR/JP는 YYYY.MM.DD, US는 MM/DD/YYYY
- 통화 표시: KRW, USD, JPY를 지원하며 환율 계산이 필요한 가격 비교는 이번 범위에서 제외
- 주소 형식: 국가별 주소 필드 순서가 달라야 한다.
- 개인정보 및 위치정보 처리: GDPR, CCPA, 한국 개인정보보호법, 위치정보법 검토 필요
- 쿠키/분석 동의: EU/UK 사용자는 비필수 분석 이벤트 수집 전 동의가 필요하다.

## 9. 접근성, 플랫폼, 성능, 보안

- 접근성 목표: WCAG 2.1 AA
- 스크린리더: iOS VoiceOver, Android TalkBack, Web NVDA 기본 탐색 지원
- 키보드 조작: 쿠폰 저장, 매장 선택, 추천 카드 탐색은 Tab/Enter로 가능해야 한다.
- 플랫폼: iOS 16 이상, Android 10 이상, Chrome/Safari 최신 2버전
- 반응형 기준: 360px, 768px, 1280px
- 성능 목표: 추천 API p95 500ms 이하, 추천 카드 첫 노출 2.5초 이하
- 보안: HTTPS 필수, 위치정보와 구매 이력은 전송/저장 시 암호화, 로그에는 개인정보 마스킹
- 인증 방식: 기존 회원 인증 세션을 사용하며 상세 정책은 확인 필요
